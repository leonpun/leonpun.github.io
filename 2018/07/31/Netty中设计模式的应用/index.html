责任终止机制

如果不调用 ChannelHandlerContext 中的 fireChannelxxx 一系列的方法，就会终止传播。默认事件是传播到下一个责任处理器的。

策略模式

例如 DefaultEventExecutorChooserFactory 类中，newChooser()方法会根据线程池的大小（是否为2的幂次）动态的选择取模运算的方式。

public final class DefaultEventExecutorChooserFactory implements EventExecutorChooserFactory {

    public static final DefaultEventExecutorChooserFactory INSTANCE = new DefaultEventExecutorChooserFactory();

    private DefaultEventExecutorChooserFactory() { }

    @SuppressWarnings("unchecked")
    @Override
    public EventExecutorChooser newChooser(EventExecutor[] executors) {
        if (isPowerOfTwo(executors.length)) {
            return new PowerOfTwoEventExecutorChooser(executors);
        } else {
            return new GenericEventExecutorChooser(executors);
        }
    }

    ······
}

接下来，next()方法就会有两种实现，分别为：

// PowerOfTwoEventExecutorChooser
@Override
public EventExecutor next() {
    return executors[Math.abs(idx.getAndIncrement() % executors.length)];
}

// GenericEventExecutorChooser
@Override
public EventExecutor next() {
    return executors[idx.getAndIncrement() & executors.length - 1];
}

单例模式

单例模式主要实现三个功能：

  • 全局唯一实例
  • 延迟加载
  • 线程安全

例如 MqttEncoder 类，私有构造函数保证只有类自己才能创建实例，同时 static final 修饰的 INSTANCE 只有类使用时才被 JVM 加载，默认会给类加上同步块保证线程安全。

@ChannelHandler.Sharable
public final class MqttEncoder extends MessageToMessageEncoder<MqttMessage> {

    public static final MqttEncoder INSTANCE = new MqttEncoder();

    private MqttEncoder() { }

    ······
}

还有例如 ReadTimeoutException 类，也是通过这种方式实现单例模式的。

public final class ReadTimeoutException extends TimeoutException {

    private static final long serialVersionUID = 169287984113283421L;

    public static final ReadTimeoutException INSTANCE = new ReadTimeoutException();

    private ReadTimeoutException() { }
}

装饰者模式

WrappedByteBuf 类的两个子类 UnreleasableByteBuf 和 SimpleLeakAwareByteBuf，构造函数中通过调用父类构造函数并重写父类方式实现装饰被装饰者父类，例如 UnreleasableByteBuf 类的 release() 方法是直接返回 false 表示不可被释放。

final class UnreleasableByteBuf extends WrappedByteBuf {

    private SwappedByteBuf swappedBuf;

    UnreleasableByteBuf(ByteBuf buf) {
        super(buf instanceof UnreleasableByteBuf ? buf.unwrap() : buf);
    }

    @Override
    public boolean release() {
        return false;
    }

    ······
}

同样的方式，装饰后可实现跟踪内存泄漏。

class SimpleLeakAwareByteBuf extends WrappedByteBuf {

    private final ByteBuf trackedByteBuf;
    final ResourceLeakTracker<ByteBuf> leak;

    SimpleLeakAwareByteBuf(ByteBuf wrapped, ByteBuf trackedByteBuf, ResourceLeakTracker<ByteBuf> leak) {
        super(wrapped);
        this.trackedByteBuf = ObjectUtil.checkNotNull(trackedByteBuf, "trackedByteBuf");
        this.leak = ObjectUtil.checkNotNull(leak, "leak");
    }

    @Override
    public boolean release() {
        if (super.release()) {
            closeLeak();
            return true;
        }
        return false;
    }

    private void closeLeak() {
        // Close the ResourceLeakTracker with the tracked ByteBuf as argument. This must be the same that was used when
        // calling DefaultResourceLeak.track(...).
        boolean closed = leak.close(trackedByteBuf);
        assert closed;
    }

    ······
}

观察者模式

ChannelFuture 作为被观察者，调用 addListener()等方法添加观察者，跟进 writeAndFlush()方法，就可以看到在 pipeline 的尾节点会创建一个 ChannelPromise 对象作为被观察者，并在它上面注册观察者。

通过 ChannelPromise 中方法的回调，就可以在观察者中获得相应的结果。
